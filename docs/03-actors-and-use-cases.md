# Акторы И Сценарии

## Акторы

| Актор | Описание | Права / Область |
|---|---|---|
| **User** | Конечный пользователь | Регистрация, логин, refresh, logout |
| **Admin** | Администратор системы | Управление ролями, блокировка пользователей, просмотр/отзыв сессий |
| **Auditor** | Роль аудита | Чтение ролей и сессий пользователей |
| **Support** | Роль поддержки | Чтение ролей и сессий пользователей |
| **Content Manager** | Роль контент-операций | Роль присутствует в auth-модели, доменные права задаются потребителями токена |
| **Client App** | Web/Mobile/Admin клиент | Передает токены и refresh токены |
| **Domain Services** | Другие сервисы | Проверяют JWT через JWKS |

## Сценарии

### Пользовательские сценарии

| Сценарий | Описание | Предусловия | Постусловия |
|---|---|---|---|
| Register | Создать учетную запись | email/phone уникальны | UserAccount создан |
| Login | Получить access/refresh | учетные данные валидны | выданы токены |
| Refresh | Обновить access token | refresh валиден | новый access токен |
| Logout | Отозвать refresh | refresh существует | refresh отозван |

### Админские сценарии

| Сценарий | Описание | Предусловия | Постусловия |
|---|---|---|---|
| Assign Role | Назначить роль | admin авторизован | роль обновлена |
| List Roles | Получить роли пользователя | actor имеет право чтения | возвращен список ролей |
| Block User | Заблокировать пользователя | user существует | доступ запрещен |
| Unblock User | Разблокировать пользователя | user заблокирован | доступ разрешен |
| List Sessions | Получить сессии пользователя | actor имеет право чтения | возвращен список сессий |
| Revoke Session | Отозвать конкретную сессию | admin авторизован | сессия отозвана (`admin_revoked`) |

### Поддерживающие сценарии

| Сценарий | Описание | Акторы |
|---|---|---|
| JWKS Publish | Публикация ключей | Auth Service |
| Token Verify | Проверка подписи JWT | Domain Services |

## Правила и ограничения

1. Access token живет коротко и не хранится в базе.
2. Refresh token живет дольше и может быть отозван.
3. Refresh token ротируется при refresh.
4. Reuse старого refresh токена приводит к отзыву всех активных сессий пользователя.
5. Заблокированный пользователь не может логиниться.
6. Роли меняются только через Admin use case.
7. Просмотр ролей/сессий разрешен `self/admin/auditor/support`.
8. Все use cases должны иметь проверку политик и аудит.
