# Auth Service — Ubiquitous Language

## Purpose

Единый язык фиксирует термины для auth‑контекста.
Он используется в коде, документации и коммуникациях.

---

## Core Terms

### UserAccount

- **Определение**: учетная запись пользователя.
- **Роль**: корень агрегата для управления учетными данными.
- **Ограничения**:
  - уникальный email/phone
  - может быть заблокирован

---

### Credential

- **Определение**: данные для входа (password, OAuth).
- **Роль**: подтверждает принадлежность учетной записи.
- **Ограничения**:
  - пароль хранится только как хеш
  - один аккаунт может иметь несколько credential

---

### Role

- **Определение**: уровень доступа пользователя.
- **Примеры**: `admin`, `user`.
- **Ограничения**:
  - роли присваиваются только через политики

---

### Session / RefreshToken

- **Определение**: долгоживущая сессия для обновления access токена.
- **Ограничения**:
  - может быть отозвана
  - неактивна после истечения срока

---

### Access Token (JWT)

- **Определение**: короткоживущий токен доступа.
- **Содержит**:
  - `sub` (user_id)
  - `roles`
  - `exp`, `iss`, `aud`

---

### JWKS

- **Определение**: JSON Web Key Set, публичные ключи подписи JWT.
- **Роль**: используется другими сервисами для проверки токенов.

---

## Forbidden Terms

| Term | Reason |
|---|---|
| Plain Password | пароль никогда не хранится в открытом виде |
| Token Store in Client | токены не хранятся в небезопасных местах |
| Admin Override | права не обходят политики |
